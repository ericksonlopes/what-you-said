## Description
Currently, WhatYouSaid is an open system with no authentication layer. To allow for secure, multi-user access (especially when hosted publicly), we need to add **Google SSO (Single Sign-On)**. This feature must be **optional** and controlled via environment variables. If enabled, the application should force login before accessing any data or features. If disabled (default), the system should remain publicly accessible.

## Tasks
- [ ] **Settings Integration**: Add `AUTH__ENABLE_GOOGLE`, `AUTH__GOOGLE_CLIENT_ID`, `AUTH__GOOGLE_CLIENT_SECRET`, and `AUTH__REDIRECT_URI` to `src/config/settings.py`.
- [ ] **Domain Entity**: Create a `User` entity in `src/domain/entities/user.py` to store basic profile information (email, name, picture).
- [ ] **Infrastructure Repository**: Implement `UserSQLRepository` in `src/infrastructure/repositories/sql/user_repository.py` for persistence.
- [ ] **Infrastructure Service**: Implement `AuthService` in `src/infrastructure/services/auth_service.py` to handle OAuth2 exchange with Google.
- [ ] **Application Use Case**: Create `AuthUseCase` in `src/application/use_cases/auth_use_case.py` to orchestrate login, logout, and session/JWT generation.
- [ ] **API Routes**: Add a new `auth_router.py` in `src/presentation/api/routes/` with `/auth/google/login` and `/auth/google/callback` endpoints.
- [ ] **API Dependencies**: Implement a `get_current_user` dependency in `src/presentation/api/dependencies.py` that validates the session/token.
- [ ] **Frontend Login**: Create a `LoginPage` component in `frontend/src/pages/Login.tsx` that appears if the user is not authenticated.
- [ ] **Frontend State**: Implement an `AuthContext` in `frontend/src/store/AuthContext.tsx` to manage user sessions and provide auth state to components.
- [ ] **Frontend API**: Update `frontend/src/services/api.ts` to include the authentication token in the headers of all requests.

## Additional Context
- Use `python-jose` for JWT handling or `fastapi-users` if a more robust solution is desired.
- Ensure that if `AUTH__ENABLE_GOOGLE` is `false`, the `get_current_user` dependency defaults to a "Guest" or bypassed state to maintain current behavior.
- The `User` entity should at least store `email`, `name`, and `picture` from the Google profile.
- Consider using an `httpx` client for the back-channel OAuth2 communication.
