# Copilot / Contributors instructions

This file guides GitHub Copilot, automated tools, and contributors on conventions, commands, and best practices for working in this repository "what-you-said".

## Overview
- Language: Python
- Main layout:
  - `src/` — source code
  - `tests/` — automated tests
  - `alembic/`, `alembic.ini` — database migrations (Alembic)
  - `main.py` — entry point
  - `pyproject.toml`, `mypy.ini`, `pytest.ini` — tool configurations

## Local setup
1. Create and activate a virtual environment (Windows):
   - python -m venv .venv
   - .\\.venv\\Scripts\\Activate
2. Install dependencies:
   - Example (uv): `uv install` and `uv sync`
   - Install locally in editable mode: `python -m pip install -e .`
3. Check `pyproject.toml` and `mypy.ini` before running checks.

## Running
- Main entry (example):
  - `python main.py` (or `python -m main` depending on the entrypoint)
- For specific tasks, consult `README.md`.

## Tests and quality
- Run tests: `pytest -q` or `python -m pytest`
- Type checking: `mypy src tests`
- Formatting: `black .`
- Import sorting: `isort .`
- Always run tests and checks before committing.

## Database migrations (Alembic)
- Apply migrations: `alembic upgrade head`
- Create a new autogenerated revision:
  - `alembic revision --autogenerate -m "description"`
- Review migration files before applying to production.

## Change rules (Copilot / contributor guidelines)
- Make surgical, focused changes; do not modify unrelated code.
- For changes that affect design, public API, or multiple files, create a plan before implementing (see "Planning").
- Update documentation and tests when changing public behavior.
- Do not commit secrets, credentials, or large binary files.
- Run `mypy` and `pytest` after changes.

## Planning and tasks (recommended flow)
- For complex changes: create a `plan.md` in the session directory (automation tools may use a session path) describing the problem, approach, and tasks.
- Use the `todos` table (when available) to track larger tasks.

## Commits and PRs
- Commit message: short subject line, blank line, explanatory body when needed.
- Required commit trailer (include in all commits):

Co-authored-by: Copilot <223556219+Copilot@users.noreply.github.com>

- Open small, focused PRs. Run CI locally when possible.

## Best practices for Copilot CLI / Automated Assistant
- Prefer small, testable, and reversible changes.
- Use `create` / `edit` for file changes (avoid creating temporary files in the repo).
- Before running commands, check the project context (`pyproject.toml`, `README.md`).
- On Windows, use backslash paths `\\`.

## Important files
- `pyproject.toml` — declares dependencies and tooling config
- `README.md` — project overview and user instructions
- `mypy.ini`, `pytest.ini` — quality/configuration files
- `alembic/` — migration history

## Checklist before committing
- [ ] All tests pass
- [ ] `mypy` reports no significant errors
- [ ] Code formatted (`black`, `isort`)
- [ ] Documentation updated (if applicable)
- [ ] Commit message includes the required trailer

## Contact
- For architecture or design questions, open an issue in the repository.
